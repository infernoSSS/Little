1) Основным управляющим классом является Game, он подключает Классы менеджеры, отвечает за частоту отрисовки и update
2) В классе Game существует метод update, внутри этого метода должны проводится ВСЕ изменения полей экзепляров классов менеджеров и нижестоящих по цепочке

3) Существует класс scene 
3.1) каждый экземпляр класса scene должен описывать одну внутриигровую сцену и обеспечивает существование и взаимодействие объектов на ней
3.2) каждый экземпляр класса scene хранится в локальном txt файле в виде набора характеристик соответствующей сцены и загружается в проект при подгрузке сцены
3.3) сцена разбита на пронумерованные квадрыты 80х80 pix, в каждом может быть расположено несколько объектов, таблица соответствия номера к-нат предаётся сюда из менеджера
3.4) Хранение : строка в txt сначала название главной музыкальной темы сцены, затем Hashmap.toString где ключи(int) - номера квадратов, а значения(int[]) - список id объектов в этом квадрате
3.5) При загрузке сцены подключается музыка из assets, а объекты загружаются в HashMap
3.5) При update сцены происходит update всех объектов, сдесьже при надобности происходит удаление объектов
3.6) При отрисовки сцены происходит отрисовка всех объектов сцены
3.7) При смене сцены должны быть выгрежены все assets из памяти
3.8) Метод scene pause

4) Существует класс менеджер SceneManager
4.1) у этого класса должен быть 1 объект.
4.2) В нём хранится HashMap список всех сцен, где ключи(int) - id сцены, а значения(String) - названия txt файла с характеристиками сцены
4.3) Здесь хранится матрица соответсвия номера квадрата и соответствующих ему к-нат.
4.4) Здесь хранится HashMap список всех не игровых объектов, где ключи(int) - id не игрового объекта, а значения(String[]) - названия текстур не игровых объектов
4.5) Здесь хранится двумерный массив смежных сцен для перехода между ними
4.6) При update sceneManager происходит update текущей сцены здесь же при надобности происходит смена сцены.
4.7) При отрисовке sceneManager происходит отрисовка сцены

5) Существует класс GameObj
5.1) От этого класса должны наследоваться все классы отвечающие за объекты
5.2) Экземпляры этого класса - не игровые объекты т.е. с ними нельзя взаимодействовать
5.3) За исключением столкновений
5.4) При создании объекта в него подгружаются его текстуры
5.5) Объект может менять свою текущию текстуру
5.6) При удалении объекта текстуры также должны удаляться
